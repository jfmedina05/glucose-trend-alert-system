# Data Folder

This folder contains synthetic input data, processed pipeline outputs, and generated summary reports for the **Glucose Trend Alert System**.

> **Important:** All data in this project is synthetic. This folder must not contain real patient data, protected health information, or data exported from an actual continuous glucose monitor.

---

## Folder Structure

```text
data/
│
├── raw/
├── processed/
└── reports/
```

---

## Purpose

The `data/` folder supports the end-to-end synthetic CGM analytics workflow:

```text
Raw synthetic data
        ↓
Risk classification
        ↓
Trend detection
        ↓
Simulated alert generation
        ↓
Processed output data
        ↓
Summary reports
```

This data structure helps separate original synthetic inputs from processed outputs and analysis reports.

---

## `raw/`

The `raw/` folder stores generated synthetic CGM-style input datasets.

These files are created by:

```text
src/glucose_alert_system/data_generator.py
```

or by running:

```bash
python src/glucose_alert_system/data_generator.py
```

Example files:

```text
data/raw/synthetic_cgm_24_hour_baseline_scenarios.csv
data/raw/synthetic_cgm_7_day_baseline_scenarios.csv
```

The raw datasets may include baseline scenarios such as:

* Stable glucose
* Rising glucose
* Falling glucose
* Post-meal spike
* Overnight drop

Example raw data columns:

| Column             | Description                              |
| ------------------ | ---------------------------------------- |
| `timestamp`        | Synthetic glucose reading timestamp      |
| `patient_id`       | Synthetic patient identifier             |
| `glucose_mg_dl`    | Simulated glucose value                  |
| `scenario`         | Synthetic glucose scenario               |
| `interval_minutes` | Simulated time interval between readings |
| `data_source`      | Synthetic data source label              |

---

## `processed/`

The `processed/` folder stores output files created after the pipeline processes raw synthetic data.

These files are created by:

```text
src/glucose_alert_system/pipeline.py
```

or by running:

```bash
python examples/run_pipeline.py
```

Example file:

```text
data/processed/processed_cgm_pipeline_output.csv
```

Processed data may include columns such as:

| Column                    | Description                             |
| ------------------------- | --------------------------------------- |
| `risk_zone`               | Simulated glucose risk zone             |
| `risk_priority`           | Simulated risk priority                 |
| `glucose_smoothed`        | Rolling-average glucose value           |
| `previous_timestamp`      | Previous reading timestamp              |
| `previous_glucose_mg_dl`  | Previous glucose value                  |
| `minutes_elapsed`         | Time between readings                   |
| `glucose_delta_mg_dl`     | Change in glucose value                 |
| `rate_mg_dl_per_min`      | Glucose rate of change                  |
| `trend_label`             | Simulated trend classification          |
| `raw_alert_type`          | Alert type before suppression logic     |
| `raw_alert_priority`      | Alert priority before suppression logic |
| `alert_persistence_count` | Consecutive alert condition count       |
| `persistence_passed`      | Whether persistence requirement was met |
| `alert_active`            | Whether final simulated alert is active |
| `alert_suppressed`        | Whether the alert was suppressed        |
| `suppression_reason`      | Reason for alert suppression            |
| `alert_type`              | Final simulated alert type              |
| `alert_priority`          | Final alert priority                    |
| `alert_message`           | User-centered simulated alert message   |

---

## `reports/`

The `reports/` folder stores summary CSV files generated by the pipeline.

Example files:

```text
data/reports/risk_zone_summary.csv
data/reports/trend_summary.csv
data/reports/alert_summary.csv
```

Report types:

| Report                  | Purpose                                                |
| ----------------------- | ------------------------------------------------------ |
| `risk_zone_summary.csv` | Counts and percentages of simulated glucose risk zones |
| `trend_summary.csv`     | Counts and percentages of trend labels                 |
| `alert_summary.csv`     | Counts of active simulated alerts by type and priority |

---

## Generating Data

From the project root, generate raw synthetic data:

```bash
python src/glucose_alert_system/data_generator.py
```

Run the full pipeline:

```bash
python examples/run_pipeline.py
```

This will create or update files in:

```text
data/raw/
data/processed/
data/reports/
```

## Recommended Git Tracking

Recommended to commit:

```text
data/raw/sample synthetic datasets
data/README.md
```

Recommended to ignore or regenerate as needed:

```text
data/processed/
data/reports/
```

Processed data and reports can be regenerated by running the pipeline, so they do not always need to be committed unless they are being used as stable examples for the README or documentation.

## Synthetic Data Policy

This repository should only contain synthetic data.

Do not add:

* Real CGM data
* Real patient identifiers
* Protected health information
* Personal health records
* Medical device exports
* Clinical datasets containing identifiable information

All patient IDs in this project should be synthetic, such as:

```text
SYNTH-001
SYNTH-002
SYNTH-003
```

---

## Important Limitations

The data in this folder is simulated.

It does not represent real glucose behavior, real patient physiology, real CGM accuracy, or clinical outcomes.

The data should only be used to test and demonstrate the software pipeline.
