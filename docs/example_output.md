## Project Name

Glucose Trend Alert System

## Purpose

This document provides representative examples of outputs generated by the synthetic CGM analysis pipeline.

All outputs are based on synthetic data only.

---

## Important Disclaimer

This project is not a medical device and is not intended for clinical use.

All glucose readings, risk zones, trend labels, and alerts shown here are simulated.

---

## Processed Data Columns

The end-to-end pipeline generates a processed CSV file containing:

| Column | Description |
|---|---|
| `timestamp` | Synthetic glucose reading timestamp |
| `patient_id` | Synthetic patient identifier |
| `scenario` | Simulated glucose scenario |
| `glucose_mg_dl` | Simulated glucose value |
| `risk_zone` | Simulated glucose risk zone |
| `risk_priority` | Simulated risk priority |
| `glucose_smoothed` | Rolling average glucose |
| `previous_timestamp` | Previous reading timestamp |
| `previous_glucose_mg_dl` | Previous glucose value |
| `minutes_elapsed` | Time difference between readings |
| `glucose_delta_mg_dl` | Change in glucose value |
| `rate_mg_dl_per_min` | Rate of glucose change |
| `trend_label` | Simulated trend category |
| `raw_alert_type` | Alert type before persistence and cooldown |
| `raw_alert_priority` | Alert priority before suppression |
| `alert_persistence_count` | Consecutive readings with same alert condition |
| `persistence_passed` | Whether persistence requirement was met |
| `alert_active` | Whether final alert is active |
| `alert_suppressed` | Whether alert was suppressed |
| `suppression_reason` | Reason for suppression |
| `alert_type` | Final alert type |
| `alert_priority` | Final alert priority |
| `alert_message` | User-centered simulated alert message |
